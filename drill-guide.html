Phishing Email — Giả mạo UBND tỉnh

INITIAL ACCESS · T1566.001 · T+0:00
🔴
Thao tác diễn tập

📋 Hành động cần thực hiện (CLIENT-PC)

  • 1. Chọn máy CLIENT-PC từ sidebar trái → tab Màn hình
  • 2. Double-click icon Outlook trên desktop để mở email client
  • 3. Trong hộp thư đến: nhấp vào email từ "UBND Quảng Ninh" (có dấu ⚠ đỏ, đính kèm file)
  • 4. Đọc nội dung email — chú ý địa chỉ người gửi và loại file đính kèm
  • 5. Chuyển tab Logs → System để xem nhật ký email gateway (bước Blue Team)

🔴 Red Team — Kỹ thuật tấn công

Kẻ tấn công đăng ký domain ubnd-tinhquangninh.info (domain giả, mới đăng ký 1 ngày). Soạn email spear-phishing nhắm mục tiêu cụ thể với nội dung liên quan đến công việc thực tế (báo cáo tài chính UBND). File đính kèm dùng double extension: .pdf.exe — Windows ẩn đuôi .exe, nạn nhân thấy file trông giống PDF.

🔵 Blue Team — Phát hiện & Phân tích

  • Kiểm tra domain người gửi: ubnd-tinhquangninh.info ≠ quangninh.gov.vn
  • SPF/DKIM check: Domain mới (1 ngày tuổi), SPF FAIL, DKIM absent
  • File attachment: .exe bị ngụy trang — email gateway nên block loại file này
  • Lệnh kiểm tra (Security team):
CLIENT
Email header analysis
# Kiểm tra header email trong Outlook: File → Properties → Internet headers
# Hoặc dùng tool online: mxtoolbox.com/EmailHeaders.aspx
Received-SPF: Fail (domain of ubnd-tinhquangninh.info)
DKIM-Signature: (none)
X-Mailer: Kali Linux / Gophish v0.12.1
# Đây là dấu hiệu email được gửi bằng công cụ phishing framework!
⚠ Dấu hiệu nhận biết: (1) Domain không kết thúc .gov.vn; (2) File .exe đính kèm trong email công vụ; (3) Nội dung tạo cảm giác KHẨN CẤP với thời hạn 16:00; (4) SPF FAIL khi kiểm tra header

Kiểm tra persistence

SecurityHealth REG_SZ %windir%\system32\SecurityHealthSystray.exe
*** WindowsDefend32 REG_SZ C:\Users\NHVIEN~1\AppData\Local\Temp\svchost32.exe
^ Registry Run key giả — chạy malware mỗi khi Windows khởi động!

🔵 Phản ứng Blue Team

  • EventID 4657: HKLM\...\Run registry value được SET — do svchost32.exe (PID 4821)

  • Khắc phục: reg delete HKLM\...\Run\WindowsDefend32 /f và schtasks /delete /tn "\Microsoft\Windows\UpdateAssistant\Assistant" /f

Cover Tracks — Xóa dấu vết

IMPACT / COVER · T1070 · T+1:35
WEB-SERVER
Red Team — Xóa log
# Kẻ tấn công xóa dấu vết trên Web Server:
rm /var/log/apache2/access.log /var/log/apache2/error.log

Biện pháp Phòng thủ theo từng bước

📧
Bước 1 — Email Phishing
Email Gateway lọc file .exe trong attachment. Cấu hình SPF/DKIM/DMARC cho domain nội bộ. Đào tạo nhân viên nhận biết phishing — kiểm tra domain trước khi mở file.
🔒
Bước 2 — Execution
Application Allowlisting (chỉ cho phép chạy file từ thư mục được phép). EDR solution (CrowdStrike, SentinelOne) phát hiện process trong %TEMP%. Chặn outbound port 4444 tại firewall.
🔑
Bước 3–4 — Persistence & Credential Dump
Sysmon ghi lại Registry modifications. Bật Credential Guard (Windows 10+) để bảo vệ LSASS. Tắt WDigest để ngăn cleartext credential lưu trong memory. Cấu hình lsass.exe run as Protected Process.
🌐
Bước 5–6 — Network Scan & Lateral Movement
Network segmentation: client PC và server ở VLAN riêng, không SSH trực tiếp được. Bắt buộc MFA cho SSH admin. IDS/IPS phát hiện và block port scan. Chính sách: không dùng password đơn giản cho account cao quyền.
🐚
Bước 7 — Webshell
Thư mục /uploads/ cấu hình php_flag engine off. File integrity monitoring (AIDE) cảnh báo khi có file mới trong webroot. WAF rule phát hiện ?cmd= trong request.
🗄️
Bước 8–9 — Exfiltration & Cover Tracks
DLP (Data Loss Prevention) theo dõi data transfer khối lượng lớn ra ngoài. MySQL general_log bật để audit query. Log forwarding realtime về SIEM — không thể xóa log từ xa. Database firewall: chỉ cho phép app server kết nối DB.
